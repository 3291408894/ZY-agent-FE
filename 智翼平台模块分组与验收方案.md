# 智翼平台 — 模块分组开发与验收方案

> 将 6 大业务模块拆为「前三个」和「后三个」两阶段，前三个完成后进行**初步验收**，验证核心闭环是否走通，再启动后三个。

---

## 二、第一阶段：前三个模块（预计 1.5 周）

### 2.1 模块范围

| 序号 | 模块 | 包含 PBI | 故事点 | 核心能力 |
|------|------|----------|--------|---------|
| **M1** | 认证模块 | PBI_01 | 18 | 注册、登录、密码找回、个人档案 CRUD、学习仪表盘 |
| **M2** | AI Agent 模块 | PBI_04, PBI_12 | 42 | 自然语言对话、任务拆解编排、工具自动调度、思考链可视化 |
| **M3** | 课文总结模块 | PBI_06 | 16 | 手动输入/粘贴课文 → AI 总结（精简/详细双模式）、知识点提取 |

> **第一阶段总故事点：76 pts**（含前端页面 + 后端 API + 联调）

### 2.2 人员分配

| 角色 | 人员 | 负责模块 | 前端工作 | 后端工作 |
|------|------|---------|---------|---------|
| **开发 A** | （后端偏重） | M1 认证 | 登录/注册/忘记密码页面、个人设置页、仪表盘页 | 认证 API、用户 CRUD API、JWT 中间件、仪表盘聚合接口 |
| **开发 B** | （前端偏重） | M2 AI Agent | Agent 对话界面、思考链面板、SSE 流式渲染、会话列表 | Agent SSE 接口、LLM 调度、工具注册框架（先接入总结工具）、会话存储 |
| **开发 C** | （全栈均衡） | M3 课文总结 | 总结输入页（文本+粘贴）、结果展示页（精简/详细切换）、历史列表 | 总结 API（SSE）、Prompt 模板、总结记录存储 |

### 2.3 模块依赖与开发顺序

```
M1 认证（最先，其他人依赖）
  │
  ├── M2 AI Agent ──────┐
  │                      │
  └── M3 课文总结 ───────┼── Agent 调用总结工具
                         │
                    交叉联调点
```

**开发节奏**：

| 天数 | M1 认证 (A) | M2 Agent (B) | M3 总结 (C) |
|------|------------|-------------|------------|
| Day 1-2 | 后端：用户表、注册/登录 API、JWT | 前端：对话界面骨架 + SSE 封装 | 后端：总结 Prompt + API |
| Day 3-4 | 前端：登录/注册页面 | 后端：Agent SSE + 工具注册框架 | 前端：总结输入/结果页 |
| Day 5-6 | 个人档案 + 仪表盘（前后端） | 思考链面板 (PBI_12 前端) | 精简/详细切换 + 历史列表 |
| Day 7-8 | **联调**：A 的认证对接 B、C | **联调**：M2 调用 M3 总结工具 | **联调**：M3 被 Agent 调用 |
| Day 9-10 | 修 bug + 完善 | 修 bug + 完善 | 修 bug + 完善 |

### 2.4 初步验收标准（Checklist）

完成以下所有项，第一阶段验收通过：

#### 功能验收

- [ ] **M1-1**：邮箱/手机号注册成功，自动创建学习档案，选择年级
- [ ] **M1-2**：登录后跳转仪表盘，显示历史记录、统计、推荐
- [ ] **M1-3**：记住密码、找回密码流程可用
- [ ] **M1-4**：个人设置（年级、学科偏好、教材版本）可修改保存
- [ ] **M2-1**：在 Agent 对话框输入自然语言（如"帮我总结《背影》"），Agent 正确识别意图
- [ ] **M2-2**：Agent 自动调用课文总结工具，返回总结结果
- [ ] **M2-3**：思考链面板实时展示 Agent 的任务拆解和工具调用步骤
- [ ] **M2-4**：思考链支持收起/展开，历史会话可回看
- [ ] **M3-1**：手动输入或粘贴课文内容，生成结构化总结（主旨+段落+考点）
- [ ] **M3-2**：精简版/详细版切换正常，内容无明显错误
- [ ] **M3-3**：历史总结记录可查看、删除

#### 非功能验收

- [ ] UI 护眼配色生效，字体三档可调
- [ ] 登录态过期后自动跳转登录页
- [ ] 核心 API 响应时间 < 2 秒
- [ ] SSE 流式输出无卡顿、无乱码
- [ ] PC 和平板布局正常（响应式）

#### 核心闭环

> **用户故事**：一名七年级学生 → 注册登录 → 在对话框输入"帮我总结《桃花源记》" → Agent 自动拆解任务并调用总结工具 → 返回精简版总结 → 学生切换到详细版查看完整考点 → **闭环走通 ✅**

## 三、第二阶段：后三个模块（预计 1.5 周）

### 3.1 模块范围

| 序号 | 模块 | 包含 PBI | 故事点 | 核心能力 |
|------|------|----------|--------|---------|
| **M4** | 文件管理模块 | PBI_05 | 16 | 多格式上传（TXT/MD/PDF/DOCX/CSV/JSON/HTML/XML/YAML）、解析、基于文件内容触发总结/出题 |
| **M5** | 习题模块 | PBI_08, PBI_09, PBI_10 | 48 | 自定义出题、做题/解析双模式、自动批改+纠错反馈 |
| **M6** | 知识图谱模块 | PBI_11 | 16 | 基于学科/知识点生成图谱、拖拽缩放、节点详情、导出图片 |

> **第二阶段总故事点：80 pts**

### 3.2 人员分配

| 角色 | 人员 | 负责模块 | 前端工作 | 后端工作 |
|------|------|---------|---------|---------|
| **开发 A** | （后端偏重） | M4 文件管理 | 文件上传页（拖拽+进度条）、文件列表、解析状态轮询 | 文件上传/解析 API、Celery 异步解析任务、多格式解析器（PDF/DOCX 等） |
| **开发 B** | （前端偏重） | M5 习题 | 出题配置页、做题模式页、解析模式页、批改结果展示 | 出题 API（SSE）、批改 API、Prompt 模板（出题+批改） |
| **开发 C** | （全栈均衡） | M6 知识图谱 | ECharts 图谱渲染、拖拽缩放交互、节点详情弹窗、导出 | 图谱生成 API、节点/边数据构建、节点详情接口 |

### 3.3 第二阶段关键集成点

```
M4 文件管理 ──── 文件解析结果 → Agent 工具注册（扩展 Agent 能力）
M5 习题模块 ──── 出题/批改 → Agent 工具注册（扩展 Agent 能力）
M6 知识图谱 ──── 图谱数据 → 可由 M4/M5 触发生成
```

### 3.4 最终验收标准

- [ ] **M4-1**：9 种格式文件全部可上传，成功率 100%
- [ ] **M4-2**：PDF/DOCX 解析文字无乱码、无缺失
- [ ] **M4-3**：上传文件后可在 Agent 中基于文件内容进行总结/出题
- [ ] **M5-1**：按学科/知识点/难度自定义出题，题型覆盖选择/填空/简答/计算/辨析
- [ ] **M5-2**：做题模式隐藏答案，解析模式展示完整解题步骤
- [ ] **M5-3**：客观题自动判分准确率 100%，主观题要点打分合理
- [ ] **M5-4**：批改响应 ≤ 3 秒，错题有纠错建议
- [ ] **M6-1**：图谱层级清晰，节点可拖拽，滚轮缩放流畅
- [ ] **M6-2**：点击节点查看知识点详情+例题
- [ ] **M6-3**：图谱可导出图片

## 四、协作约定

### 4.1 接口文档更新

- 每个模块启动前，负责人先在本规范文档中**补充/确认该模块的 API Schema**
- 开发 B（Agent 模块）负责维护 **Agent 工具注册接口**，M3/M4/M5 的工具需要按统一格式注册

### 4.2 Agent 工具注册接口（M2 提供给其他模块）

```typescript
// 其他模块接入 Agent 时需实现的工具描述格式
interface AgentTool {
  name: string;           // 工具唯一标识，如 "text_summary"
  description: string;    // 给 LLM 看的描述，用于意图匹配
  parameters: {           // JSON Schema 格式
    type: "object";
    properties: Record<string, { type: string; description: string }>;
    required: string[];
  };
  handler: string;        // 后端 API 路径，Agent 将调用此接口
}
```

| 阶段 | 提供方 | 工具名称 | 说明 |
|------|--------|---------|------|
| 第一阶段 | M3 开发 C | `text_summary` | 课文总结 |
| 第二阶段 | M4 开发 A | `file_parse` | 文件解析 |
| 第二阶段 | M5 开发 B | `exercise_generate` | 习题生成 |
| 第二阶段 | M5 开发 B | `exercise_grade` | 习题批改 |
| 第二阶段 | M6 开发 C | `knowledge_graph` | 知识图谱生成 |

### 4.3 每日站会（15 分钟）

- 每人同步：昨天做了什么、今天计划、有无阻塞
- 阻塞项优先解决，特别是**跨模块依赖**（如 M2 等待 M3 的工具接口）
