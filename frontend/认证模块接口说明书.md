## 一、通用约定

### 1.1 统一响应格式

所有接口返回统一的 JSON 结构：

**成功响应：**

```json
{
  "code": 0,
  "message": "ok",
  "data": { ... }
}
```

**错误响应：**

```json
{
  "code": 40001,
  "message": "邮箱已被注册",
  "detail": null
}
```

### 1.2 鉴权方式

需要登录的接口，前端必须在请求头携带 JWT Token：

```
Authorization: Bearer <access_token>
```

Token 由登录接口返回，存储在 `localStorage` 中。

### 1.3 请求 Content-Type

- 所有 POST/PUT 接口使用 `Content-Type: application/json`
- 文件上传接口使用 `multipart/form-data`（认证模块不涉及）

// 通用类型

/** 统一成功响应包装 */

  page: number
  page_size: number

// 用户相关类型
// ============================================================

  id: string
  email: string | null
  phone: string | null
  nickname: string
  grade: string | null           // 如 "七年级"
  subjects: string[]             // 如 ["语文", "数学", "英语"]

  avatar_url: string | null

  id: string
  email: string | null
  phone: string | null

  grade: string | null
  subjects: string[]
  textbook_version: string | null

/** 仪表盘数据 */

  total_study_time: number      // 累计学习时长（秒）
  total_exercises: number       // 累计做题数

}

// ============================================================
// 请求体类型
// ============================================================

/** 注册请求 */
interface IRegisterReq {
  email: string | null          // 邮箱（与 phone 至少填一个）
  phone: string | null          // 手机号（与 email 至少填一个）
  password: string              // 密码，8-64 位
  grade: string                 // 年级，如 "七年级"（必填）
  subjects: string[]            // 学科偏好列表，如 ["语文", "数学"]

/** 登录请求 */

  login: string                 // 邮箱或手机号
  password: string              // 密码

interface IUpdateProfileReq {

  grade?: string
  subjects?: string[]
  textbook_version?: string

  email: string | null

## 三、认证接口（/auth）

### 3.1 用户注册

```
POST /api/v1/auth/register
```

> **无需登录态**

**请求体：**

```json
{
  "email": "student@example.com",
  "phone": "13800138000",
  "password": "Abc123456!",
  "grade": "七年级",
  "subjects": ["语文", "数学", "英语"]
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `email` | string \| null | 否 | 邮箱，与 phone 至少填一个 |
| `phone` | string \| null | 否 | 手机号，与 email 至少填一个 |
| `password` | string | **是** | 密码，长度 8-64 位 |
| `grade` | string | **是** | 年级，如 "七年级"、"八年级" |
| `subjects` | string[] | 否 | 学科偏好列表，默认 `[]` |

**成功响应 (HTTP 201)：**

```json
{
  "code": 0,
  "message": "注册成功",
  "data": {
    "user_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "email": "student@example.com",
    "grade": "七年级"
  }
}
```

**可能错误：**

| code | 说明 | 处理建议 |
|------|------|----------|
| 40901 | 邮箱/手机号已被注册 | 提示用户更换或去登录 |
| 40001 | 参数校验失败（如密码太短、年级为空） | 提示具体错误信息 |

**前端注意事项：**
- 注册成功后**不会自动登录**，需要跳转到登录页让用户手动登录
- 注册时会自动创建学习档案（`learning_profiles` 表），初始数据全为 0

### 3.2 用户登录

```
POST /api/v1/auth/login
```

> **无需登录态**

**请求体：**

```json
{
  "login": "student@example.com",
  "password": "Abc123456!"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `login` | string | **是** | 邮箱或手机号（二选一，后端自动匹配） |
| `password` | string | **是** | 密码 |

**成功响应 (HTTP 200)：**

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "token_type": "bearer",
    "expires_in": 86400,
    "user": {
      "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
      "email": "student@example.com",
      "phone": "13800138000",
      "nickname": "小明",
      "grade": "七年级",
      "subjects": ["语文", "数学"],
      "textbook_version": "部编版",
      "avatar_url": null
    }
  }
}
```

**可能错误：**

| code | 说明 | 处理建议 |
|------|------|----------|
| 40001 | 账号或密码错误 | 提示"账号或密码错误"（不要区分是账号还是密码错） |
| 40401 | 用户不存在 | 同上，统一提示"账号或密码错误" |

**前端注意事项：**

- `access_token` 需要存入 `localStorage`，key 为 `access_token`
- `expires_in` = 86400 秒 = **24 小时**（由后端 `JWT_ACCESS_TOKEN_EXPIRE_MINUTES` 配置决定，默认 1440 分钟）
- `user` 对象需存入 Pinia Store（`useUserStore`），供全局使用
- 登录成功后跳转到仪表盘页 `/dashboard`

### 4.3 学习仪表盘

```
GET /api/v1/users/dashboard
```

> **需登录态**

**请求参数**：无

**成功响应 (HTTP 200)：**

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "total_study_time": 36000,
    "total_exercises": 128,
    "correct_rate": 0.85,
    "recent_summaries": [],
    "recent_exercises": [],
    "recommendations": [],
    "weak_points": ["文言文阅读", "二次函数"]
  }
}
```

**响应字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `total_study_time` | number | 累计学习时长（秒），前端需转换为"X小时Y分钟" |
| `total_exercises` | number | 累计做题数量 |
| `correct_rate` | number | 正确率，0.0 ~ 1.0，前端需转为百分比显示如"85%" |
| `recent_summaries` | array | 近期总结（Sprint 1 前期为空数组） |
| `recent_exercises` | array | 近期习题（Sprint 1 前期为空数组） |
| `recommendations` | array | AI 推荐内容（Sprint 1 前期为空数组） |
| `weak_points` | string[] | 薄弱知识点列表 |

**前端注意事项：**
- `correct_rate` 为 0~1 的小数，显示时需 `(correct_rate * 100).toFixed(0) + '%'`
- `total_study_time` 单位为秒，格式化示例：
  ```typescript
  const hours = Math.floor(totalStudyTime / 3600)
  const minutes = Math.floor((totalStudyTime % 3600) / 60)
  ```
- 仪表盘在三阶段（近期/习题/推荐）数据为空时有合理的空状态 UI

## 五、Token 管理

### 5.1 存储

登录成功后，Token 存储在 `localStorage`：

```typescript
localStorage.setItem('access_token', loginData.access_token)
```

### 5.2 请求拦截器自动附加

```typescript
// api/request.ts
request.interceptors.request.use((config) => {
  const token = localStorage.getItem('access_token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})
```

### 5.3 Token 过期处理

Token 有效期 **24 小时**（1440 分钟）。过期后后端返回 `40101` 或 `40102`。

响应拦截器统一处理：

```typescript
request.interceptors.response.use(
  (res) => res.data,
  (error) => {
    if (error.response?.status === 401) {
      localStorage.removeItem('access_token')
      // 清除用户状态
      const userStore = useUserStore()
      userStore.clearProfile()
      // 跳转登录页
      router.push('/login')
    }
    ElMessage.error(error.response?.data?.message || '请求失败')
    return Promise.reject(error)
  }
)
```

### 5.4 路由守卫

前端需在 Vue Router 中配置全局前置守卫：

```typescript
// router/index.ts
router.beforeEach((to, from, next) => {
  const token = localStorage.getItem('access_token')
  const publicRoutes = ['/login', '/register', '/forgot-password']

  if (!token && !publicRoutes.includes(to.path)) {
    next('/login')
  } else if (token && publicRoutes.includes(to.path)) {
    next('/dashboard')
  } else {
    next()
  }
})
```

### 5.5 Token 格式

JWT Token 结构（HS256 签名）：

```
Header:  { "alg": "HS256", "typ": "JWT" }
Payload: {
  "sub": "user-uuid",       // 用户 ID
  "iat": 1751539200,        // 签发时间（Unix timestamp）
  "exp": 1751625600,        // 过期时间（签发时间 + 1440 分钟）
  "type": "access"          // Token 类型
}
```

### 6.3 Pinia Store 示例

```typescript
// src/stores/user.ts
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'
import type { IUserBrief, IUserProfile } from '@/types/auth'
import { getProfile, updateProfile } from '@/api/modules/auth'

export const useUserStore = defineStore('user', () => {
  const profile = ref<IUserBrief | null>(null)
  const isLoggedIn = computed(() => !!profile.value)

  /** 设置用户信息（登录成功后调用） */
  function setUser(user: IUserBrief) {
    profile.value = user
  }

  /** 获取完整资料 */
  async function fetchProfile() {
    const res = await getProfile()
    profile.value = res.data
  }

  /** 更新资料并刷新本地状态 */
  async function updateUserProfile(data: {
    nickname?: string
    grade?: string
    subjects?: string[]
    textbook_version?: string
  }) {
    await updateProfile(data)
    await fetchProfile()
  }

  /** 清除登录态（退出登录或 Token 过期时调用） */
  function clearProfile() {
    profile.value = null
  }

  return {
    profile,
    isLoggedIn,
    setUser,
    fetchProfile,
    updateUserProfile,
    clearProfile,
  }
})
```

## 七、错误码速查

认证模块涉及的错误码：

| code | HTTP Status | message | 说明 | 前端处理 |
|------|-------------|---------|------|----------|
| 0 | 200/201 | ok | 成功 | 正常处理 data |
| 40001 | 400 | 参数校验失败 | 请求参数不合法 | 提示用户修正输入 |
| 40101 | 401 | Token 过期 | JWT 已超过有效期 | 清除 Token，跳转登录页 |
| 40102 | 401 | Token 无效 | JWT 签名验证失败 | 清除 Token，跳转登录页 |
| 40301 | 403 | 无权操作该资源 | 无权限 | 提示"无权限" |
| 40401 | 404 | 用户不存在 | 用户 ID 无效 | 提示并跳转 |
| 40901 | 409 | 邮箱/手机号已被注册 | 注册冲突 | 提示用户更换或去登录 |
| 42901 | 429 | 请求频率超限 | 触发频率限制 | 提示用户稍后再试 |
| 50001 | 500 | 服务器内部错误 | 未知服务器错误 | 提示"服务器异常，请稍后再试" |
